В данном примере хотел привести в качестве закрытого класса какой-нибудь утильный класс,
но решил немного больше почитать и привел пример на основе статьи из википедии:

""Идея была в том, что однажды разработанная реализация класса в дальнейшем требует только исправления ошибок,
 а новые или изменённые функции требуют создания нового класса.
 Этот новый класс может переиспользовать код исходного класса через механизм наследования.
 Производный подкласс может реализовывать или не реализовывать интерфейс исходного класса.""

 Кажется довольно сложным выбрать правильный момент для оставления модуля открытым к расширению,
 помимо случаев, где надо исправлять ошибки.
