В качестве абстрагирование хотел бы привести рабочий пример. Был класс обработки данных экрана из "карусели",
он был универсальный для различных типов обработки, но со временем стало понятно, что количество экранов увеличивается
и добавляется не заявленный ранее функционал, в связи с чем был сделан абстрактный класс и выполнено наследование для
нескольких типов экранов.

В качестве факторизации можно привести тоже рабочий пример, где у нас был отдельный класс для вызова какого-то "тула"
для неиросети из стороннего репозитория и обработки openApi документации для разбора и вызова указанного эндпоинта
из загруженного документа. Со временем это обросло дополнительной функциональностью и все классы для работы с инструментами
были добавлены в иерархию Tool с выносом общей логики.
