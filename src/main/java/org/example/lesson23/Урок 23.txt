В качестве примера можно взять иерархию Самокат -> Электросамокат.
Пусть у самоката будут методы ride и park для начала поездки и парковки соответственно. В случае с самокатом мы можем выполнять
эти методы без предусловий, но электросамокат может наложить ограничение поскольку для начала поездки есть предусловие
активации самоката, а для парковки наличие необходимого места. В результате метод scooter.ride() может выбросить
неожиданную ошибку или вернуть неожиданный код результата.

В качестве примера постусловия предлагаю взять иерархию карта -> кредитная карта. Где возможна ситуация, в которой при
пополнении карты через ее метод в случае кредитной карты постусловие может быть не "средства внесены на счет", а
"средства внесены на счет за вычетом долга за месяц", что может оказать неожиданный эффект при дальнейшей работе с балансом.
